Blocks Module

block

<?php [echo] get("blocks.block", $block_name, [$site_id]); ?>

Description

Returns the block text for the given $block_name, $site_id (defaults to current site), and the current $_SESSION['language']. The data is returned (rather than echo'd) in case it needs to be assigned to a variable. To display it directly, you will need to put an echo statement before the get() command.

This tag provides functionality that is as close to a standard CMS as we have. It allows for blocks of HTML to be written to the database and displayed on a page. The block can contain an entire page, or just a portion of it.

Blocks can have the same Block Name but different languages so that you can create multi-language pages this way. Block Names must be unique within a site ID.

Template Variables

This tag does not use a template.

page

<?php get("blocks.page", $page_name, [$site_id]); ?>

Description

Returns an array containing the page metadata for the given $page_name, $site_id (defaults to current site), and the current $_SESSION['language']. The data is returned (rather than echo'd) since it will need to be assigned to a variable.

This tag and it's associated database table are intended to extend the block tag and allow for the block tag to be used to store whole pages. The idea is that you would create a page entry and block entry with the same Page/Block Name and use them to build the whole page. Language strings would be used to change language info in the header and footer.

Template Variables

This tag does not use a template.
