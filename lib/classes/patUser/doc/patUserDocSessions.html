Sessions simplified
Sessions are used to store data for the current user. As patUser works with PHP's built-in session management, you may decide where your data is stored (filesystem, shared mem,...). patUser registers one global variable (define the name of the var when instatiating patUser), which is used as an array.
Using patUsers API, you may store any variables in this array and do not need to access the sessions with the PHP functions yourself.
API documentation
These functions are those available for the above features:

bool storeSessionValue

store a value in the session

may only be used if patUser was called with $useSessions = true
Parameters:
string$namename of the value to store

mixed$valvalue to store

Returns:

bool$success true if the value could be stored, false otherwise


mixed getSessionValue

get a value that was stored in the session

may only be used if patUser was called with $useSessions = true
Parameters:
string$namename of the value to retrieve

Returns:

mixed$val value that was stored
