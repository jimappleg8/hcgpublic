A good way to start
On a general basis, we'd recommend creating two PHP files that take care of common instructions like including all the extenal classes needed in your project. We normally name them "prepend.php" (for all instructions that have to be done before you can start), and an "append.php" which could be used for statistical functions as well as the history features of patUser.
The prepend.php in detail
 
01 include( "include/patDbc.php" );
02 include( "include/patUser.php" );
03 include( "include/patTemplate.php" );
04
05 $myDbc = new patMySqlDbc( "localhost", "mydb", "myuser", "mypass" );
06 $tmpl  = new patTemplate();
07 $tmpl->setBasedir( "templates" );
08 
09 $user = new patUser( true );
10 $user->setAuthDbc( $myDbc );
11 $user->setTemplate( $tmpl );
12
13 $user->setAuthTable( "users" );
14 $user->setAuthFields( array( "passwd" => "userpass" ) );
Lines 01 to 03: include necessary files
Here we include the patDbc class (for the mySql database connectivity), the patUser class (guess for what...) and the patTemplate class (for the output).
Lines 05 to 07: Instantiate helper classes
patDbc needs to be instantiated with all the details necessary for the connection to your user database. The patTemplate class only needs to know in which folder the templates used for the login screen are stored.
Lines 09 to 11: Startup of the user class
Here we instantiate the patUser class, and set the references to the dbc and template objects. The patUser object accepts two parameters on startup, the first defines if PHP sessions should be used, and the second defines the name of the global variable used to store all session data (default name is "patUserData").

Working with sessions
If you want to work with sessions, don't forget that you always have to add the session ID to every link. If you'd like to know more on the subject, you should take a look at the PHP manual.

Lines 13 and 14: Setting data sources
With setAuthTable(), we set the name of the table (in your database that you provided on line 05) that is used to store your main user data. With setAuthFields(), you set the names of the required user fields (primary, username and passwd) that are used to identify a given user. Default field names are "uid", "username" and "passwd".
The append.php in detail...
...is fairly simple, so we won't go into much detail - as this is mainly about the history functions of patUser, you should have a look at the section about those for more in-depth information.

These functions are those available for the above features:

Parameters:
[boolean$useSessions = true]flag to indicate that sessions should be used

[string$sessionVar = "patUserData"]name of the global var used for sessions
