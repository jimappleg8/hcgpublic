Working with user data
patUser lets you manage user data distributed over several table, even several databases on several hosts. To use this feature you have to add patDbc objects to patUser. The best way to do this is in your prepend.php. Just include and modify the following lines: 
$newDbc = new patMySqlDbc( "newhost", "newDB", "newUser", "newPass" );
$user->addDbc( "newDbc", $newDbc );
These two lines create a new database connectivity (you should fill in you hostname, dbname, username and password) and add it to the patUser object with the internal name "newDbc". It can later be referenced with this name.
Working with several tables
To work with data in other tables than the table used for authentication, you have to tell patUser how to access the tables. You should include this in your prepend.php to ensure that you have access to these tables in all you files. This is a short example of how you may add a table: 
$user->addTable( "bookmarks", array( "dbc"        => "newDbc"
                                     "table"      => "userBkmarks",
                                     "foreign"    => "uid",
                                     "primary"    => "id",
                                     "minentries" => 0,
                                     "maxentries" => 5 ) );
This will add a table with the internal name "bookmarks". It can be accessed through the connectivity "newDbc", that we defined above. The name of the table in the database is "userBkmarks" and the user id (so you know which user made which bookmark) is stored in the field "uid". As each user is allowed to have more bookmarks, you will need a primary key in your table to identify each line. You have to tell patUser, which field stores the primary key. With the last two lines you can define the minimum and maximum amount of entries in this table for each user.
You may add as many dbcs and tables as long as you use unique names.
Reading the data
As you now told patUser, where data is stored, you will want to access the data. patUser provides a powerful API for this task: 
$data = $user->getUserData( array( "table"  => "bookmarks",
                                   "uid"    => 5,
                                   "fields" => array( "url", "title" )
                            array( "url"    => "http://www.php-tools.de" ) );
This will get data from the table you added with the internal name "bookmarks". It will fetch the fields "url" and "title" (you may get any fields from your table). As a user id is specified with "uid" => 5, it will get the data for the user with the user id 5. If no user id is specified the user id of the authenticated user is taken.
With the second parameter, you may defined options for the where statement. In the above example, only these lines from the table bookmarks will be fetched where the field "url" contains the value "http://www.php-tools.de".
The function will return an array, which contains as much associative arrays as lines where found.
Changing the data
patUser provides a function that lets you modify any data of the user. 
$user->modifyUser( array( "title"   => "PHP Application Tools",
                          "url"     => "http://www.php-application-tools.de"  ),
						  
                   array( "table"   => "bookmarks",
                          "mode"    => "update",
                          "olddata" => array( "url" => "http://www.php-tools.de" ),
                          "uid"     => 5
                        )
                 );
modifyUser() is a quite powerful function, so we'll explain all parameters here. The first array specifies the new data that should be stored in the database.
The second parameter specifies a lot of options. At first you define the table in which you want to change data. Here you also define the mode of the change: do you want to "insert" new data or "update" existing data. If you are modifying existing data, you'll need some values to identify the old entry, if there could be more than one entry for each user in the table. Use "olddata" to specify which entry you would like to edit. If you want, you may specify a user id; if it's left out, the user id of the authenticated user is used.
The above example would change the bookmark for http://www.php-tools.de for the user with the user id 5. It will change the title to "PHP Application Tools" and the url to "http://www.php-application-tools.de".
Deleting data
Of course it's also possible to delete users or data of users: 
$user->deleteUser( array( "table"   => "bookmarks",
                          "olddata" => array( "url" => "http://www.php-application-tools.de" ),
                          "uid"     => 5
                        )
                 );
This example will delete the bookmark for http://www.php-application-tools.de of the user with user id 5. If no user id is given, the currently logged in user is used. If no "olddata" is specified, all entries in the defined table will be deleted. And finally if no table is given all data for the user will be deleted.
Adding new user
patUser provides a function that enables you add a new user to your database. 
$user->addUser( array( "username" => "foo",
                       "passwd"   => "bar",
					   "email"    => "foo@bar.de" ),
                true );
This will add a user with the name "foo" to your database. The user has the eMail adress "foo@bar.de" and the password "bar". As the second parameter is set to true, the user will automatically be logged in.

Popular "retype password" feature
If you want to use the popular "retype password" feature, when new users are registering on your webpage, just supply an array with two entries as passwd parameter for the addUser() function. patUser will automatically check, if both passwords are identical or return an error code otherwise.

addUser() will return the user id of the new user if he could be added or false otherwise.
Building user lists
As you'll probably have more than one user on your site, you will want to create user lists. patUser provides a function, which can gat lists for you: 
$list = $user->getUsers( array( "uid", "username", "email" ),
                         array( array( "field" => "name"
                                       "value" => "a",
                                       "match" => "startswith" ),
                         array( "orderby" => "name",
                                "limit"   => 10 )
                        );
The first parameter specifies the names of the fields you like to select from the database.
The second contains a list of conditions for the select statement. See Building conditions for details.
The last parameter specifies further options for the list. Available options are: table (to define the table, from which the fields are fetched), orderby (to define the field by which the result is ordered, limit (if you want to limit the amount of returned list) and offset (if you do not want the first users in the list). Furthermore you can use "conditions" => "all" or "any" to specify wether the users have to match all or any of the specified conditions.
The above example will return user id, name and email of the first 10 users from the authentication table, whose name starts with an "a" and sorts them by name
