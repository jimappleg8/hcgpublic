Getting field values without knowing where they are
patUser helps you making your application flexible, when user data is involved. Imagine you are working on an application that needs the eMail adress of the user, that is currently logged in.
Normaly you would store this information in a predefined table with a predefined name. Using patUser you may allow the user of your application to choose, where he stores the eMail adress. All he has to do is to configure the patUser object correctly (using the setFieldLocation() function) and hand it over to your application.
To get the eMail adress you just have to ask patUser for the value of the field "email". patUser will build and send the query to the correct database and return the result to your application

void addFieldLocation

set the location of a field

Parameters:
string$namename of the field

[string$table = "authTable"]name of the table that stores the field (use "authTable" if its stored in the authtable)

[string$field = ""]name of the field in the table (leave out if identical with name)



mixed getField

fetch the value of a field

Parameters:
string$fieldname of the table

[integer$uid = 0]user id, if no uid is givven the authenticated user will be used

Returns:

mixed$value value of the field
