Finding users amongst the lot
Using the getUsers() function you can select users via several conditions. But there is one limitation to getUsers(): all conditions hav to work on the same table. With the searchUsers() function, we got rid of this limitation and added some new features. As the function as a bit tricky, we'll try to explain how it works.
How does it work?
Generally spoken searchUsers() works like getUsers(), but there are some differences:


Only fetches user ids
searchUsers() will always fetch only the user ids of the found users. If you need more data on the users, you will have to use getUserData(), for each user that was found.
Because of this, searchUsers() will always return an array, which just contains integer values instead of arrays.


define a table for each condition
If you are not familiar with creating conditions, you should read the section on Building conditions first.
When using searchUsers() you have to add the name of the table to each condition array (use "table" => "myTable").


counting table entries
You may also count entries in a table for a certain user. Just set the "extra" parameter of a condition to "countentries". 
$condition1 = array( "table" => "userBookmarks",
                     "extra" => "countentries",
                     "value" => 3,
                     "match" => "greater" );
This condition will return all users, that have at least three bookmarks.
Available options
The options parameter has to be an associative array, which contains all option names as keys and option values as value. As of now, there is only one option available: you may choose if you want a list of users that match all or any of your conditions.
Complex example
 
$conditions    = array();
// as the auth table should be used, to table has
// to be defined
$conditions[0] = array( "field" => "count_logins",
                        "value" => 50,
                        "match"	=> "greater" );

$conditions[1] = array( "table" => "bookmarks",
                        "extra"	=> "countentries",
                        "value"	=> array( 1, 5 ),
                        "match"	=> "betwen" );
						
$users = $user->searchUsers( $conditions, array( "conditions" => "any" ) );
$users will contain all ids of the users that at least logged in 50 times or have between one and five entries in the bookmarks table.
