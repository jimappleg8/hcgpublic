Using patUser's history feature
patUser helps you keeping track of which pages the user previously visited. It kind of emulates a browser history, that can easily be accessed.
You can use feature to inlude "back" links in your page or display a list of the last visited pages to the user, which helps him navigating.
Modifying your append.php
To enable history funtions you have to tell patUser to keep track of the user history. The best way to do this is to put the following line of code in your append.php, which should be included in every file of your site. 
$user->keepHistory( 5, $pageTitle );

Sessions must be enabled
In order to work with the history features, patUser must be instantiated with true as first parameter, so it works with PHP sessions.

Working with the history
With this line of code patUser will always store the last 5 pages the user visited and will add the current page with the title stored in $pageTitle.
From now on you can easily acces the history with the two functions getHistory() (which returns an array with titles and urls of the last pages the user visited) and goHistory() (which redirects the user to a page he previusly visited).
Take a look at the API documentation below for more information.
API documentation
These functions are those available for the above features:

bool keepHistory

let patUser create a history of visited pages

needs session support
Parameters:
int$amountsize of the history

[string$title = ""]title of the current page

Returns:

bool$success true if the history could be stored, false otherwise


array getHistory

get the history for the current user

Parameters:
[integer$pages = 0]number of the last pages to get

Returns:

array$history array containing titles and urls of the history


void goHistory

go back x pages in the history

as Header( "Location:..." ) is used, there must not be any output before this is called
Parameters:
[integer$pages = -1]number of the pages to go back (use negative values)
