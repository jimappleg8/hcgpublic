Organizing users in groups makes it easy to assign a permission to several users at once.

Defining tables
To work with groups, you have to tell patUser on startup, where group data is stored. Of course the best way is again, to include the following lines in your prepend.php: 
01 $user->setGroupTable( "groups" );
02 $user->setGroupFields( array( "primary" => "gid", "name" => "name" ) );
03 $user->setGroupRelTable( "usergroups" );
04 $user->setGroupRelFields( array( "uid" => "uid", "gid" => "gid" ) );
In line 01 you define the table, that stores the group data. In line 02 you define the fieldnames of the group table. Required is a primary key (default is "gid") and a name for each group.
To organize users in groups, you need a table that stores user-group relations. This table consists of two fields: one that stores a user id and one that stores a group id. In lines 03 and 04 you define the name of the table and how you called the fields in the table.
Working with groups
As you've definded the tables and finished setup, you may use groups in your website. patUser provides a lot of functions for these tasks. 
$groups = $user->getGrous( array( "gid", "name" ),
                           array( array( "field" => "name"
                                         "value" => "pat",
                                         "match" => "contains" ) ),
                           array( "orderby" => "name" )
                         );
This function is similar to the getUsers() method. Take a look at Working with user data for more information.
Modifying, adding and deleting groups
Similar to the functions for user data, we've implemented functions for working with groups. These functions are in particular:


getGroupData()
This function retrieves data of a group.


modifyGroup()
This function modifies a group. The first parameter is used to identify the group and the second parameter is the new data for the group.


addGroup()
Use this function to add a group. It accepts an assoc array as the only parameter. This array at least has to contain a name for the group.


deleteGroup()
Use this function to delete a group. It accepts an assoc array as the only parameter. This array is used to identify a group, the easiest way to delete a group is supplying the group id.
Organizing users in groups
With patUser you can organize users in group. We implemented several functions that allow you to do this. 
$user->addUserToGroup( array( "uid" => 5, "gid" => 10 ) );
This will add the user with the user id 5 to the group with the group id 10.

Getting IDs
Use identifyUser() and identifyGroup() to get user or group ids. 

$user->removeUserFromGroup( array( "uid" => 5, "gid" => 10 ) );
This will remove the user with the user id 5 from the group with the group id 10.
Which user is in which group?
Of course you will want to know which user is in which group. patUser supplies three functions to retrieve this information:


getJoinedGroups()


getUsersInGroup()


isMemberOfGroup()
Take a look at the API documentation below for more information.
