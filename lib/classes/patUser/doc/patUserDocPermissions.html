Permission Management made easy
patUser helps you managing permissions the make certain parts of your website only accessable to certain users. Of course patUser stores the permissions in a MySQL table. The structure of the table should at least contain these fields:
fieldname fieldtype description
id integer field This field either stores the user or group id for the permission
id_type enum('user','group') This field defines wether the id stored in the previous field references a user or a group.
perms set('read','delete','modify','add') This is were the permissions are stored.
Of course you may change the name of the fields, but be sure to set the names correctly using patUsers() setPermTable() and setPermFields() functions.
Using this table structure you are able to tell patUser that for example user with the user id has read access or may add anything. As you see permissions can only be added globally. So you should at at least one field to the table, that stores the part of your website for which the permission is set. You may again use setPermFields() to do this (we advise you to do this in your prepend.php). 
$user->setPermFields( array( "id"      => "id",
                             "id_type" => "id_type",
                             "perms"   => "perms",
                             "part"    => "part" )
                    );
This will use the default names for the required fields (id, id_type and perms) and tell patUser that there is an additional field, which stores any detail for the permission.
Adding and deleting permissions
For adding and deleting permissions there are two functions:


addPermission()


deletePermission()
To add a permission, you just call addPermission() which accepts only an associative array as the only parameter. A typical function call could look like this: 
$user->addPermission( array( "id_type" => "user",  // adding permission for a user
                             "id"      => 5,       // user id is 5
                             "part"    => "news"   // part of the site
                             "perms"   => array( "read", "delete" )
                            )
                    );
With this line you would grant the user read and delete permission for the news part of your site. If you want to grant him access to only some of the news, you could possibly add another field to the permFields using setPermFields() an call it topic...
Deleting permissions works exactly the same ways and thus we won't explain it here. You could easily remove the 'delete' permission and the 'read' permission would remain in the database.
Deleting all permissions for a user or group in a part
The only values required when calling the deletePermission() function are id_type and id. If you leave out the other parameters, all permissions of the user will be deleted. If you call it with array( "id_type" => "user", "id" => 5, "part" => "news" ) all permission the user 5 had in the news part will be deleted.
Getting and checking permissions
patUser supplies two more functions that are used in the permission management. One is to get all or certain permissions the other can be used to check wether a user is allowed to do something.
To get all permissions of a user, you should use this command: 
$perms = $user->getPermissions( array( "id_type" => "user",
                                       "id"      => 5 ) );
To get the permissions the user has in a certain part use: 
$perms = $user->getPermissions( array( "id_type" => "user",
                                       "id"      => 5 )
                                       "part"    => "news" );
getPermissions() accepts a second parameter, which enables you to use a feature that is seldomly needed. By default, getPermissions also fetches the permissions of all groups the user has joined and adds them to the user permission. When setting the second parameter to "user", only the permission directly related to the user will be returned, when using "group" only the permissions of the groups he has joined will be returned.
To check wether a user has the right to add news, you may use this function: 
$user->hasPermission( "id_type" => "user",
                      "id"      => 5,
                      "part"    => "news",
                      "perms"   => array( "add" ) );
If the user has the permission to add news, this function will return true - otherwise it will return false.

Using global permissions
When setting global permissions (they consist only of id_type, id and perms), they are valid for all parts or action that you have in your application. So the above example will also return true, if you added "add" permission for the user 5 without specifying a part.

hasPermission accepts two additional parameters: the second specifies wether the user needs to have all permissions specified or if one permission is sufficient. With the third you may tell patUser, that group permissions of the joined groups should not be fetched....
API documentation
These functions are those available for the above features:

void setPermTable

set tablename that contains permissions

Parameters:
string$tablename of the table that contains permissions



void setPermFields

set fieldnames that are used in the permissions

Parameters:
array$fieldsassoc array containing fieldnames of the permission table



bool addPermission

add permission(s)

Parameters:
array$permarray containing the permission(s) to be added

Returns:

bool$success true if the permission could be added, false otherwise


bool deletePermission

delete permission(s)

Parameters:
array$permarray containing the permission(s) to be deleted

Returns:

bool$success true if the permission could be deleted, false otherwise


array getPermissions

get all permissions of a user

Parameters:
[array$clause = array()]params for the permissions (any fields in your permission table)

[string$type = "both"]get user or group permissions or both ( both|user|group )

Returns:

array$perms permissions of the user


boolean hasPermission

check, wether user has a permission

Parameters:
[array$perm = array()]permission(s) to be checked

[string$mode = "all"]all or any permission (all|any)

[boolean$includeGroups = true]should group permissions be checked,too?

Returns:

boolean$hasPermission true, if the user has the permission, false otherwise
