Chapter 6. Assigning values to form fields

One of the biggest differences between a standard "smarty" form and a formsess form is the way you assign data to your form fields.

In a smarty form, a field value is handled using smarty variables: you create a variable in the appropriate field in the template, and assign a value to that variable from your PHP Script 

TEMPLATE

<form method="POST" action="foo.php" name="bar">
  <input type="text" name="foo" value="{$FOO}" />
<form>

      

SCRIPT

$smarty->assign('FOO', 'bar');

Such a method has several weakness:

  • logic separation: forms are usually part of the backend logic, since what is posted using the forms is sent to scripts. This way a lot of stuff depends on the template designer.
  • sometimes complex: in some situations, such a method can require a lot of template code; by instance for a checkbox, a select, a radio button... this is why the {html_*} smarty functions were written, but they don't fit all the situations

Formsess helps you avoiding these problems. You create your form in your template without thinking about the assigned values. All that stuff is handled from the script, and you don't even have to type a single line of smarty code. 

TEMPLATE

<fs:form method="POST" action="foo.php" name="bar">
  <fs:input type="text" name="foo" />
</fs:form>

      

SCRIPT

$fs = new formsess('bar');
$fs->assign('foo', 'bar');

And it works for any type of form field: select, radio, checkbox, text, hidden... You no longer have to think about quotes escaping, loops to check for a checkbox value. Read the documentation for the assign method for more information
