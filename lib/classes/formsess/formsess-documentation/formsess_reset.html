reset()

void reset($target);
string $target;

Very important function. Since data is stored by formsess using the session, their lifetime depends on the session lifetime. Then it is quite likely that you will end up with useless data, and you might even get them in a place where you didn't expect them. reset is there to avoid these problems.

Just call reset when you need to clear up the data you no longer need. By instance, before performing the validation on a form, it is quite likely that you want to clear the errors that occured during the last run. Then just call formsess::reset('errors'). If you have data saved in a form and no longer need it _ by instance after an insert in a database _ just call reset without any parameter to clear any information related to that form.

During the developement, you are probably gonna change your forms several times: add or remove fields, add or remove validation steps... formsess is not aware of what you are changing. If you add a validation step in your template, it will be stored in the session data space, and will be executed the next time you call performCheck, EVEN if the <fs:validate /> tag was removed. To clear up the session space, you have to use reset. Just add it once and remove it from your PHP code, and you're done. 


<?php
$fs = new formsess('foo');
$fs->reset();
?>

So if you feel like you're getting a very very strange behaviour from your form, it is quite likely that a little reset is needed.

The following values are valid for $target:

  • values
  • errors
  • types
  • all (default)
